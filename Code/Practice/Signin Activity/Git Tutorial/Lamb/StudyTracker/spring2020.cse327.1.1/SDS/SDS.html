Introduction

STUDY POLICE is a mobile app that will track the reading state and time of a reader. It is especially useful to faculty/teachers since he/she can easily track his/her students. It can provide information about how long a student spends looking at the study material. Also, the app shows the read time of several parts of a single study material Our main goal is to keep track of every student's progress. It identifies the face of a specific person using face recognition service. User would need to sign up and sign in to do these activities, third party log in system would be available also.
This document will provide a specific design for the software and also will provide a implementation plan for the developers working under the project.

Description of Design Components

This application is composed of three major components. They are:
1. Classroom
2. Profile Creation and Authentication
3. Study Under Surveillance

1. Classroom: This component together with its sub-components will work as an online classroom where instructors will be able to create new classrooms, upload material and manage classrooms. Students will be able to join the classrooms and read materials. Instructors will also get information regarding the reading time of each student and reading time of each material.
This component is composed of three sub-components. They are:
1.1 Create Class
1.2 Join Class
1.3 Manage Class

1.1 Create Class: This sub-component will be able to create a new classroom in which students will be able to join with a specific classroom code.

1.2 Join Class: With this sub-component, students will be able to join a classroom by a classroom code.

1.3 Manage Class: With this sum-component, instructors will be able to manage his/her classroom. Here he/she will be able to upload materials, track students and get information regarding reading time of the students and reading time of the materials.

This sub-component is composed of further more sub-components. They are:
1.3.1 Upload Materials
1.3.2 Track Students
1.3.3 Get Info

1.3.1 Upload Materials: By this sub-component, instructors will be able to upload new materials that the students will read.

1.3.2 Track Students: By this sub-component, instructors will be able to track the students of his/her class.

1.3.3 Get Info: By this sub-component, instructors will be able to get information regarding the reading time of each student and the reading time of each material.

2. Profile Creation and Authentication: This component will act as an profile creation and authentication system where users will be able to create profiles and will be able to log in with proper authentication. This is composed of two sub-components. They are:
2.1 Create Profile
2.2 Log In

2.1 Create Profile: This sub-component will act as an profile creation/Registration system where users will be able to create profile and add information to their profile.

2.2 Log In: This sub-component is a login system where users will be able log in to his/her account.

3. Study Under Surveillance: This component will be composed of both facial recognition and eye tracking and together it will act as a component that will be able to recognise the students through facial recognition and will tract their actual reading time through eye tracking. Its subcomponents are:
3.1 Facial Recognition
3.2 Eye Tracking

3.1 Facial Recognition: This sub-component will recognise the students with facial recognition. Through valid identification, students will be able to add to his/her reading time. Facial recognition API will be used for this component.

3.2 Eye Tracking: This sub-component will use eye tracking to measure the reading time of the students and also the reading time of a reading material. Eye tracking API will be used for this component.

Database: All the components together will interact with the database. Database will consist of the data regarding classroom, user profiles, materials, face ID, eye tracking data, facial recognition info and all the information regarding each component.

Database Design

ER Diagram

Table Description

Relational Schema

User (User_Id , name, gender, email, password)

Class (Class_Id, user_id, class_name, access_code)

Student (User_Id, class_id)

Face_info (Face_Id, user_id)

Materials (Material_Id, class_id, link,study_hour)

User - This table stores information about users in the database. User can be a faculty or student or both.

Attributes

  • User_Id User_id is the primary key of this table. It uniquely identifies each user.
  • Name Name is compound attribute of this table as it can be divided in first name ,middle name and last name. Name will be set by the user. It will be visible to the users and will help to identify or search a user easily.
  • Gender Gender is a single valued attribute. This will help to mention a user as male or female when another user replies on his/her comment.
  • E-mail E-mail is an attribute. A user can open one account only with a specific email id.
  • Password - Password is a single valued attribute. If somehow system fails to recognize a face, a user can login using password.

Class This table stores information about classes opened by the instructors.

Attributes

  • Class_id - Class_id is the primary key of this table. It uniquely identifies each class.
  • User_id User_id is a foreign key came from user table. It will identify a user.
  • Class_name Class_name is a compound attribute. Each class will have a given class name.
  • Access_code Access_code is a single valued attribute. A user will require a specific access code to join a specific class.

Student This table is for all the users who are currently enrolled in a class as a student. This will show which user is enrolled in which class as a student.

Attributes

  • User_id - User_id is a foreign key came from user table. It will identify a user.
  • Class_id - Class_id is a foreign key came from class table. It uniquely identifies each class.

Face_info This table stores all the information regarding face recognition information.

Attributes

  • Face_id It is a primary key of this table to uniquely identify a users information regarding face recognition.
  • User_id - User_id is a foreign key came from user table. It will identify a user.

Materials This table stores all the information of the study materials.

Attributes

  • Material_Id It is the primary key of this table and uniquely identifies each study material.
  • Class_id - Class_id is a foreign key. It uniquely identifies each class.
  • Link Link is an attribute for the study materials.
  • Study_hour Study_hour is a single attribute to show total time of study of a user.
